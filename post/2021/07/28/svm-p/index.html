Support Vector Machines: Preliminaries

Lagrange duality and kernel methods

Yuhan Wang
2021-07-28
5 min

In this article, Lagrange duality and kernel methods are discussed as preliminaries for understanding the basics of the SVMs.

# Lagrange duality

Consider an optimization problem of the following form:

wmin​f(w)

s.t. gi​(w)hi​(w)​≤0, i=1,2,…,k=0, i=1,2,…,l​

To solve this problem, we define the generalized Lagrangian:

L(w,α,β)=f(w)+i=1∑k​αi​gi​(w)+i=1∑l​βi​hi​(w).

Let's define a function θP​(w) as

θP​(w)=α,β:αi​≥0max​L(w,α,β).

If w does not satisfies the constraints of the optimization problem above, it is clearly that

θP​(w)=α,β:αi​≥0max​(f(w)+i=1∑k​αi​gi​(w)+i=1∑l​βi​hi​(w))=∞.

But if w satisfies the constraints, then θP​(w)=f(w). As a result, it is not difficult to find that the following problem(called the primal problem) is equivalent to the original optimization problem:

wmin​θP​(w)=wmin​α,β:αi​≥0max​L(w,α,β).

Now we define another function

θD​(α,β)=wmin​L(w,α,β).

The so-called dual problem is as the following form:

α,β:αi​≥0max​θD​(α,β)=α,β:αi​≥0max​wmin​L(w,α,β).

The optimal values of the primal problem and the dual problem are denoted by p∗ and d∗. It is apparent that

d∗=α,β:αi​≥0max​wmin​L(w,α,β)≤wmin​α,β:αi​≥0max​L(w,α,β)=p∗.

Suppose f and gi​ 's are convex, and hi​'s are affine. Moreover, gi​'s are feasible. Under these conditions we have p∗=d∗, making it possible for us to solve the dual problem instead of the primal problem. Therefore, there must exist w∗,α∗,β∗ such that w∗ is the solution to the primal problem and α∗ and β∗ are the solution to the dual problem. So p∗=d∗=L(w∗,α∗,β∗). More importantly, w∗, α∗ and β∗ satisfy the following conditions.

# Karush-Kuhn-Tucker (KKT) conditions

∂wi​∂​L(w∗,α∗,β∗)∂βi​∂​L(w∗,α∗,β∗)αi∗​gi​(wi∗​)αi∗​gi​(w∗)​=0,  i=1,2,…,d=0,  i=1,2,…,l=0,  i=1,2,…,k≥0,  i=1,2,…,k≤0,  i=1,2,…,k​

Conversly if some w∗, α∗ and β∗ satisfy the KKT conditions, then it is also a solution to the primal or the dual problem.

# Kernel methods

A kernel function K(x,z) is a map: X×X→R satisfying K(x,z)=⟨ϕ(x),ϕ(z)⟩ for any x,z∈X. ϕ is a feature map.

It is a common idea that we can transform the training instances by applying feature maps to achieve better performance. But if we do transform the vectors of the training set, the computational complexity of the training would probably increase dramatically since the dimension of the transformed vectors can be unexpectedly large. But kernels can make the whole process much more computationally efficient.

If we apply the feature map ϕ to the training instances, then the dual problem will contain the inner product ⟨ϕ(x(i)),ϕ(x(j))⟩. We can replace this inner product of transformed vectors simply by the kernel corresponding to the feature map ϕ. So, we don’t actually need to transform the training instances. The transformation is implicitly represented by the kernel. The result will be the same as if we had gone through the trouble of transforming the training set then fitting a linear algorithm.

# Commmon kernels

  • Linear: K(x,z)=xTz.
  • Polynomial: K(x,z)=(xTz+1)p.
  • Gaussian: K(x,z)=exp(−2σ2∥x−z∥2​).

# Mercer's theorem

Suppose K is a map: X×X→R. Then K is a valid kernel if and only if the kernel matrix of any {x(1),…,x(n)} is symmetric positive semi-definite.
